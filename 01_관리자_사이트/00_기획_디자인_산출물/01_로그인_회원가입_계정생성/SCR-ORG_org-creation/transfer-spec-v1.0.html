SCR-TRF기관관리자 이관 기능명세

기관 상세 페이지에서 관리자를 다른 운영자로 교체하는 기능. 기존 관리자는 운영자로, 선택된 운영자는 관리자로 역할이 교체됩니다.
CS 담당자 1차 기관관리자 2차 기관관리자
v1.0 | 2026-03-26 | 최초 작성
1
화면 개요
항목내용
이관 실행 가능 역할CS 담당자 / 1차 기관관리자 / 2차 기관관리자 (각 역할의 이관 가능 범위는 섹션 4 참고)
진입 경로기관 리스트 → 기관명 클릭 → 기관 상세 페이지(SCR-ORG-005) → 관리자 정보 영역 "이관하기" 버튼
목적기관의 관리자를 같은 기관 소속 운영자 중 한 명과 역할 교체 (관리자→운영자, 운영자→관리자). 다른 기관 간 교체는 불가.
교체 결과기존 관리자 → 운영자로 전환 / 선택된 운영자 → 관리자로 전환
용어 정의:
· 이관 — 관리자 역할을 다른 사람에게 넘기는 것. 기존 관리자와 선택된 운영자의 역할이 서로 교체됩니다.
2
이관 플로우
기관 상세 페이지에서 "이관하기" 버튼 클릭부터 이관 완료까지의 흐름입니다.
Step 1. 이관하기 버튼 클릭
구성요소유형상세
이관하기 버튼 버튼 기관 상세 페이지 관리자 정보 영역 옆에 위치.
관리자가 등록되어 있을 때만 활성화. 관리자 미등록 시 비활성.
Step 2. 경고 안내 모달
이관하기 버튼 클릭 시 경고 모달이 표시됩니다.
구성요소유형상세
경고 아이콘 아이콘 ⚠️
경고 제목 텍스트 "기관관리자 이관 안내"
경고 내용 텍스트 "관리자를 이관하면 다음과 같이 변경됩니다:"

· 현재 관리자 {관리자이름}님은 기관운영자로 전환됩니다.
· 선택한 운영자가 새로운 기관관리자로 지정됩니다.

"이관을 진행하시려면 아래에 이관하겠습니다를 직접 입력해주세요."
확인 문구 입력 텍스트 입력 안내 문구: "이관하겠습니다"를 입력하세요.
정확히 "이관하겠습니다"를 입력해야 확인 버튼 활성화. 대소문자/띄어쓰기 정확히 일치해야 함.
취소 버튼 보조 버튼 모달 닫힘
확인 버튼 주요 버튼 "이관하겠습니다" 미입력 또는 불일치 시 비활성.
정확히 입력 완료 시 활성화 → 클릭 시 운영자 선택 화면으로 이동
Step 3. 운영자 선택
해당 기관 소속 기관운영자 리스트에서 새 관리자를 선택합니다.
구성요소유형상세
모달 제목 텍스트 "새 관리자 선택" + 기관명 표시
운영자 리스트 테이블 해당 기관 소속 기관운영자만 표시
컬럼: 선택(라디오) / 이름 / 이메일 / 전화번호 / 부서
활성 계정만 표시 (비활성/만료 계정은 선택 불가)
검색 텍스트 입력 이름 또는 이메일로 검색. 실시간 필터링.
취소 버튼 보조 버튼 모달 닫힘. 이관 취소.
이관 확인 버튼 주요 버튼 "이관 완료" 클릭 → 운영자 미선택 시 비활성.
선택 후 클릭 → 역할 교체 실행.
운영자가 없는 경우:
· "이관 가능한 운영자가 없습니다." 안내 표시
· "이관 완료" 버튼 비활성
· 운영자 계정을 먼저 생성해야 이관 가능
Step 4. 이관 완료
처리 항목상세
기존 관리자 역할: 기관관리자 → 기관운영자로 전환
선택된 운영자 역할: 기관운영자 → 기관관리자로 전환
이관 이력 이관 기록 저장 (일시, 이전 관리자, 새 관리자)
성공 메시지 "관리자가 {새관리자이름}님으로 이관되었습니다."
완료 후 이동 기관 상세 페이지로 돌아감 (관리자 정보 갱신)
3
이관 이력
기관 상세 페이지 관리자 정보 하단에 최근 3건의 이관 이력을 표시합니다.
컬럼컬럼명상세
①이관 일시YYYY-MM-DD HH:MM
②이전 관리자이전 관리자 이름 (이메일)
③새 관리자새 관리자 이름 (이메일)
④처리자이관을 실행한 사람 (CS 담당자 또는 기관관리자)
이력 표시 규칙:
· 최근 3건만 표시 (최신순)
· 이관 이력이 없으면 "이관 이력이 없습니다." 표시
· 이력은 삭제 불가 (자동 관리)
4
이관 실행 권한
누가 어떤 기관의 관리자 이관을 실행할 수 있는지 정의합니다. 이관은 항상 같은 기관 내에서만 이루어집니다 (A기관 관리자를 B기관 운영자로 교체하는 것은 불가).
이관 실행자이관 가능 대상 기관교체 방식
CS 담당자 전체 기관 선택한 기관의 관리자 ↔ 같은 기관 소속 운영자 교체
1차 기관관리자 본인 소속 1차 기관 + 하위 2차 기관 해당 기관의 관리자 ↔ 같은 기관 소속 운영자 교체
2차 기관관리자 본인 소속 2차 기관 본인(관리자) ↔ 같은 2차 기관 소속 운영자 교체
※ 2차 기관관리자가 직접 이관 시 본인이 운영자로 전환됨
핵심 규칙:
· 이관은 반드시 같은 기관 소속 내에서만 가능 (다른 기관 간 교체 불가)
· 이관 대상: 해당 기관의 활성 상태 운영자만 선택 가능 (비활성/만료 계정 불가)
2차 기관관리자 본인 이관 시 주의:
· 2차 기관관리자가 본인 기관의 이관을 실행하면 본인이 운영자로 전환됨
· 이관 후 관리자 메뉴에 접근 불가 (운영자 권한으로 전환)
· 이 내용은 경고 안내 모달에서 사전 안내됨
5
성공/에러 메시지
5-1. 성공 메시지
동작메시지완료 후 이동
이관 완료관리자가 {새관리자이름}님으로 이관되었습니다.기관 상세 페이지 (정보 갱신)
5-2. 에러/안내 메시지
조건메시지
관리자 미등록 상태에서 이관 시도등록된 관리자가 없습니다. (버튼 비활성)
소속 운영자가 없음이관 가능한 운영자가 없습니다.
운영자 미선택 상태에서 이관 확인"이관 완료" 버튼 비활성 (에러 메시지 없음)
6
전체 플로우차트
